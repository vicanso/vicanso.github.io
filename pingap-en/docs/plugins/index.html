Plugin Overview and Index

Pingap's plugin system is the embodiment of its core extension capabilities. By combining different plugins, you can add rich functionalities to your APIs and services—such as authentication, security protection, performance optimization, and traffic control—without modifying Pingap's core code.

How It Works​

  1. Independent Configuration: Each plugin's configuration is stored in a separate TOML file (e.g., plugin.jwt.toml, plugin.cache.toml), keeping it separate from the main configuration for easy management.
  2. Applied to a Location: In the location.toml file, you can apply one or more plugin instances to a specific routing rule using the plugins array.
  3. Sequential Execution: Plugins in the plugins array are executed sequentially in the order they are declared. This means you can carefully arrange the execution order of plugins to implement complex logic chains (for example, apply IP restriction first, then JWT authentication).

Available Plugin List​

Below is a list of all official plugins currently supported by Pingap. Click "View Details" to get the complete configuration parameters and usage examples for each plugin.

PluginCore FunctionConfiguration Docs
Accept Encoding PluginAdjusts the Accept-Encoding request header before forwarding a request to an upstream service.View Details
Basic Auth PluginAdds HTTP Basic Auth authentication based on username and password to your services.View Details
Cache PluginCaches responses from upstream services to dramatically improve the performance of frequently accessed APIs and effectively reduce the load on backend services.View Details
Compression PluginAutomatically performs dynamic compression on the response body sent to the client, based on the client's support.View Details
CORS PluginAdds CORS support to your services, allowing clients from other domains to access your services.View Details
CSRF PluginAdds CSRF protection to your services to prevent cross-site request forgery.View Details
Directory PluginTransforms a Location in Pingap into a powerful static file server, serving files directly from the disk without forwarding requests to an upstream backend.View Details
IP Restriction PluginProvides access control based on IP addresses, supporting both allow and deny modes, and can be configured with single IP addresses or CIDR network ranges.View Details
JWT PluginProtects your APIs and services by validating JSON Web Tokens (JWT). The plugin supports multiple token delivery methods, signing algorithms, and can convert backend authentication logic into stateless JWT tokens.View Details
Key Auth PluginProvides a simple API key authentication mechanism, supporting fetching authentication information from request parameters (query) or headers. Multiple valid keys can be configured for easy multi-system access.View Details
Rate Limit PluginProtects your upstream services from traffic spikes and malicious attacks by limiting the request rate or concurrent connections per unit of time, ensuring service stability and fairness.View Details
Mock PluginDirectly returns preset HTTP responses from Pingap without needing a backend service. A powerful tool for frontend development, API debugging, simulating exceptions, and conducting load tests.View Details
Ping PluginAdds a health check endpoint to your service.View Details
Redirect PluginEasily implement forced HTTP to HTTPS redirection or add a uniform prefix to URL paths. Ensures the original request method and body are preserved after the redirect by sending a 307 Temporary Redirect response.View Details
Referer Restriction PluginImplements access control based on the origin of requests by checking the HTTP Referer header. Effectively prevents hotlinking of resources like images and videos, and can restrict API calls to trusted websites only.View Details
Request ID PluginAttaches a unique identifier to each incoming request for distributed tracing, log correlation, and troubleshooting, a key component in enhancing system observability.View Details
Response Headers PluginDynamically adds, sets, removes, renames, or conditionally sets HTTP response headers before sending the response to the client. Used to enhance security, control caching, and add custom metadata.View Details
Sub Filter PluginPerforms real-time find-and-replace on response body content. Supports simple string substitution and powerful regular expression replacement for dynamically modifying HTML, JSON, or other text content.View Details
Traffic Splitting PluginDynamically routes traffic to different upstream services based on a specified percentage. A powerful tool for implementing A/B testing, gradual rollouts, and canary releases, with support for cookie-based session persistence.View Details
User Agent Restriction PluginImplements access control for specific clients, crawlers, or scanners by inspecting the HTTP User-Agent header. Supports powerful regular expression matching and can be configured in blacklist or whitelist mode.View Details
